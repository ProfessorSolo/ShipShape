## 🚀 What We'll Learn

We’re going from “wait… how does this app actually get deployed?” to “yep, I shipped that”:

- 🐳 **Containers with Docker** — images, containers, Dockerfiles, and repeatable environments
- 🧩 **Multi-Service Development** — Docker Compose, networks, volumes, and app + database workflow
- ☁️ **Deployment** — preparing apps for production and getting them live on the internet
- 🔐 **Configuration** — environment variables, secrets, and sane production setup
- 📦 **Delivery Workflow** — the path from laptop to running service
- 👀 **Production Awareness** — logging, monitoring, versioning, and deployment history

---

## 🧠 The Mindset

This isn’t “memorize DevOps.”  
It’s learning how modern application delivery _works_ — so you can reason about packaging, deployment, and runtime behavior with confidence.

We’ll use:

- 🤖 **AI-assisted learning** (Watts still clocks in on time)
- 🧱 **Hands-on builds** — small steps, real deployment outcomes
- 🧭 **Practical patterns** — workflows that make sense outside the classroom too

---

## 🔧 Prerequisites

Bring:

- A code editor
- Basic comfort with JavaScript and Node.js
- Git/GitHub familiarity
- A willingness to break stuff… and then ship it cleanly 🔥
