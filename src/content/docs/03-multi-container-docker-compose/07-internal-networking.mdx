One of the most powerful features of Docker Compose is what it does silently the moment you run `up`. It doesn't just start containers; it builds a private, virtual infrastructure for them to live in.

## 1. The Walled Garden

When Compose starts your stack, it automatically creates a single, private bridge network. It then seamlessly attaches every service defined in your `compose.yaml` to this network.

This creates a "Walled Garden." Your containers can see and talk to each other freely, but they are isolated from the rest of your computer (the Host) unless you explicitly open a gate (using `ports`).



## 2. Service Discovery (DNS)

In our manual lab, we had to remember that our database was named `db_service`. In a professional Compose environment, we don't need to track IP addresses. Docker includes an **embedded DNS server**.

Because you named your service `db_service` in the YAML, Docker treats that name as a valid web address (a hostname) within the internal network.

## 3. Why `localhost` is a Trap

We’ve seen this before, but it bears repeating: **Inside a container, `localhost` always means the container itself.**

* If your API tries to connect to `localhost:27017`, it is looking for MongoDB inside the API's own walls.
* By using the service name `db_service`, the API "reaches out" over the internal bridge to find the correct vessel.

<NoteBox title="Minimizing the Attack Surface">
  Notice that in our `compose.yaml`, we didn't map any ports for the `db_service`.
  
  ```yaml
  db_service:
    image: mongo:7.0
    # No ports mapping here!
  ```
  
  This is a professional best practice. Since the API and the Database are in the same "Walled Garden," the API can talk to the DB on port `27017` internally. We only use `ports` on the API because we need to access it from our laptop's browser. Keeping your database port "closed" to the outside world is a massive security win.
</NoteBox>

<CautionBox title="Port Mapping vs. Internal Ports">
  * **Internal Port:** `27017` (Used by the API to talk to the DB inside the garden).
  * **Published Port:** `3000:3000` (Used by you to talk to the API from your browser).
</CautionBox>
